## Notes

## Adoption and the J-Curve

- The speaker frames AI tool adoption as following a J-curve: performance and outcomes initially dip before potential long-term gains appear.
- Most organizations are currently at the bottom of that J-curve, experiencing this early dip.
- Instead of viewing the dip as a transitional cost, many interpret it as proof that AI tools do not work or that vendors misrepresented their value.
- Because workflows have not yet adapted, the resulting inefficiencies get misread as fundamental flaws in AI rather than growing pains.

## GitHub Copilot as a Case Study

- GitHub Copilot is presented as a clear example of this dynamic.
- It reportedly has 20 million users and around 42% market share among AI coding tools, indicating wide adoption.
- Lab studies on Copilot show developers completing isolated coding tasks 55% faster when using the tool.
- These lab results are compelling for metrics and slide decks, suggesting strong productivity gains in simplified scenarios.

## Lab Gains vs. Production Reality

- In production environments, the situation is described as “much more complicated” than the lab findings.
- AI-assisted coding is associated with larger pull requests, which directly increases the amount of code that must be reviewed.
- Larger pull requests and AI-generated patterns raise review costs, as reviewers must spend more time understanding and validating the code.
- The source notes that AI-generated code introduces more security vulnerabilities, adding risk and downstream remediation costs.
- Developers are actively struggling with how to use AI coding tools well, indicating ongoing adaptation rather than settled best practices.

## The Cost Tradeoff: Writing vs. Owning Code

- A senior engineer captures the tradeoff succinctly: Copilot makes writing code cheaper but owning it more expensive.
- "Writing cheaper" refers to faster initial authoring and completion of code snippets.
- "Owning more expensive" encompasses maintenance, review, debugging, and security management over the code’s life.
- The speaker emphasizes that this sentiment is common across many engineers and applies broadly to AI-generated code, not just Copilot.
- Overall, the source portrays a tension between impressive short-term speedups in controlled tasks and increased long-term complexity and costs in real-world software development.
