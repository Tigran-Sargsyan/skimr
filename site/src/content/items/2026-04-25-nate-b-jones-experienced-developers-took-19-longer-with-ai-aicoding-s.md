## Key Points

- Organizations sit in the bottom of a J-curve, misreading early AI productivity dips as failure.
- Many leaders interpret unadapted workflows as evidence that AI tools are hype and ineffective.
- GitHub Copilot has around 20 million users and a 42% AI coding-tool market share.
- Lab studies report Copilot users completing isolated coding tasks 55% faster.
- In production, AI-generated code often leads to larger pull requests that are harder to review.
- Review costs rise because generated code increases volume and complexity requiring human oversight.
- AI-generated code introduces more security vulnerabilities that must be detected and fixed later.
- Engineers report that AI makes writing code cheaper but long-term ownership and maintenance more expensive.

## Notes

## The J-Curve and Misinterpretation of AI Performance

- The speaker describes AI adoption following a J-curve, where performance initially dips before improving.
- Most organizations are currently in the bottom part of that J-curve with AI coding tools.
- Leaders are misinterpreting this early dip as proof that AI tools do not work.
- They conclude vendors misled them and that lack of immediate gains means AI is just hype.
- Instead of seeing that workflows and practices have not yet adapted, they treat current friction as final evidence against AI.

## GitHub Copilot as a Case Study

- GitHub Copilot is presented as a clear illustration of this dynamic.
- It has very large adoption, with about 20 million users.
- The tool reportedly holds a 42% market share among AI coding tools.
- Lab studies on Copilot show striking performance numbers, such as 55% faster code completion for isolated coding tasks.
- These metrics are appealing for slide decks and marketing, suggesting large efficiency gains.

## Production Reality vs. Lab Results

- The speaker stresses that production use of AI coding tools is much more complicated than lab results indicate.
- In real-world codebases, developers see larger pull requests when using AI-generated code.
- Larger pull requests increase the effort required for code review, raising review costs.
- There is evidence of more security vulnerabilities being introduced through AI-generated code.
- These vulnerabilities must be found and corrected later, adding hidden downstream costs.
- Developers are actively struggling with how to use AI coding tools effectively in practice.

## The Cost of Writing vs. Owning Code

- A senior engineer summarises the tradeoff: “Copilot makes writing code cheaper, but owning it more expensive.”
- This captures a broader sentiment the speaker has heard repeatedly from engineers across the industry.
- The concern applies not just to Copilot but to AI-generated code in general.
- Short-term gains come from faster initial code generation and task completion.
- Long-term costs rise because the resulting code is harder to maintain, review, and secure over time.
- The core tension is between apparent productivity at the moment of typing and the real cost of owning the codebase over its lifecycle.
